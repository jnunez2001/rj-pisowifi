Network Mode
How customers get internet access
Before switching to MikroTik mode:
  • Router must run RouterOS 7+ with the REST API enabled (IP → Services → www, port 80 by default).
  • The Hotspot service must already be set up on the interface below, with a DHCP server bound to it (bandwidth limits are applied by IP, looked up from the client's DHCP lease — a device that hasn't received an IP yet can't be shaped).
  • This mode grants access by adding a bypass IP binding for the paying device's MAC, not by logging it into MikroTik's own Hotspot page — customers never see MikroTik's login screen.
  • Session timing is still enforced by this server, not the router — the router only reflects allow/block state when told to.
The interface EAP225 is connected to on MikroTik.
